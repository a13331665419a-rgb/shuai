# AI真人短剧分镜拆解 — 元流程 (Meta Process)

> 这不是某部剧的规则，而是**从任何一个剧本出发，产出一套可批量执行的分镜规则**的通用流程。
> Veronica's Sinister Spell 的 `VSS_StoryboardRules.md` 就是用这个流程产出的。

---

## 全流程概览

```
输入: 一个完整剧本（任何题材）
输出: 该剧专属的 StoryboardRules.md + 前1-3集的标杆分镜 + 可批量复用的prompt模板

流程: 
  理解剧本 → 问校准问题 → 做一集标杆 → 从标杆中归纳规则 → 用规则批量生产
```

---

## Phase 1: 剧本理解 — "这是什么？"

### 输入
- 完整剧本文件（PDF/FDX/文本）

### 要做的事
1. **通读全本**，不拆镜，只提取结构性信息：
   - 总集数、每集平均页数/场景数
   - 主要角色（几个？关系？）
   - 场景类型分布（室内多还是外景多？对话多还是动作多？）
   - 特殊制作需求（VFX？大群戏？特殊场地？）
   - 卡点/paywall位置（如果有）
   - 情绪曲线走向（开篇什么调？高潮在哪？）

2. **盘点已有资产**（如果有的话）：
   - 角色表？故事梗概？场景库？
   - 已有的东西不重复做

### 输出
一段结构性摘要，类似：
> "64集，94页。魅魔题材，GL主线。2个主角+1个主反派+若干配角。VFX需求重（变身、魔法、吸魂）。
> 场景以室内为主。情绪曲线：性感悬疑开场 → 校园欺骗 → 超自然对抗 → 感情线收尾。Paywall在Ep13/14间。"

### 为什么这步重要
不理解全局就拆第一集，会导致第一集的调性和后续集脱节。你需要知道这部剧的"终点"才能设计"起点"的镜头语言。

---

## Phase 2: 校准问题 — "用什么做？做到多细？什么风格？"

### 输入
- Phase 1 的结构性摘要

### 要问的问题（3-4个，不多问）

| # | 问题 | 为什么要问 | 影响什么 |
|---|------|---------|---------|
| 1 | **受众是谁？核心期待是什么？** 目标观众画像 + 他们来看什么 | 决定"为谁拍"，直接影响镜头的共情对象和爽点设计 | 视角选择、景别分布、角色镜头语法、beat高潮点设计 |
| 2 | **用什么AI生成工具？** | 不同工具的prompt格式完全不同 | 提示词模板结构 |
| 3 | **分镜精度？** 关键帧级 vs 镜头级 vs 混合 | 决定工作量和产出密度 | shot数量、表格详细度 |
| 4 | **视觉风格参考？** 给1-2个影视参考 | 锁定光线、色调、镜头风格 | 全部提示词的尾部风格标签 |

**为什么"受众和核心期待"要第一个问？**

这个问题决定的不是"拍什么"（剧本已经决定了），而是"怎么拍" — 同一个场景，给男性观众拍和给女性观众拍，镜头语言完全不同：
- **视角归属：** 摄影机代表谁的眼睛？观众认同谁？
- **爽点设计：** 什么时刻给反应镜头？什么时刻给power shot？
- **身体镜头的意图：** 是male gaze的展示，还是女性力量的彰显？
- **反派镜头的功能：** 是建立威胁让观众紧张，还是建立"活该被收拾"的情绪让反转更爽？

### 输出
一组确认参数，如：
> 工具=Seedance 2.0 / 精度=镜头级 / 风格=Jennifer's Body / 本session直接开始

### 为什么这步重要
不问就做 = 大概率返工。但也不能问太多 — 制作人自己也不确定全部细节，需要看到第一个样品才能给反馈。3-4个问题刚好够锁定大方向。

---

## Phase 3: 标杆制作 — "做一集给我看"

### 输入
- 第一集的剧本文本
- Phase 2 确认的参数
- 已有的角色表/场景库（如果有）

### 思考路径（这就是你问的"prompt是什么"）

拆一个场景时，脑子里跑的是这个顺序：

```
1. 读完整个场景的剧本文本
   ↓
2. 问自己：这场戏的戏剧功能是什么？
   - 建立角色？推进情节？制造反转？情感高潮？
   ↓
3. 找到情绪/权力的转移点，把场景切成beat
   - "谁在掌控？" → 掌控权交接 = beat分界
   - 每个beat给一句"导演意图"
   ↓
4. 每个beat内，逐shot设计：
   a) 这个shot存在的理由是什么？（观众需要看到什么信息？）
   b) 用什么景别？（远→看环境，近→看情绪，特写→看细节）
   c) 摄影机在哪？（和角色的关系 — 仰视=power，俯视=弱势，平视=中性）
   d) 和上一个shot是什么剪辑关系？（延续/递进/跳切/揭示）
   ↓
5. 给每个shot写AI生成提示词
   - 结构：[景别] of [人物外观+穿着], [动作/状态], [光线/氛围], [风格]
   - 30-60 words
   - 人物用外观描述不用名字
   ↓
6. 标记关键帧 + 写导演笔记
   - KF = 这个场景必须"画对"的画面
   - 导演笔记 = 给执行者的补充说明（节奏、表演、VFX优先级）
```

### 实际给Claude的prompt（如果要复用）

```
请从导演视角，将以下剧本场景拆解为镜头级分镜脚本。

【剧本文本】
{粘贴场景文本}

【背景信息】
- 这部剧的类型/调性：{类型}
- 视觉风格参考：{参考片}
- AI生成工具：{工具名}
- 这个角色的外观：{角色描述}

【你的任务】
1. 读完场景，找到情绪/权力的转移点，把场景划分为beat（每beat写一句导演意图）
2. 每个beat内设计4-8个shot，每shot包括：景别、运镜、镜头描述、角色动作、台词、VFX/SFX、AI生成提示词、备注
3. AI生成提示词格式：[景别] of [人物外观描述+穿着], [动作], [光线], [风格标签]，30-60 words，不用角色名字
4. 标记2-4个关键帧（这场戏必须画对的画面）
5. 写导演笔记：节奏控制、表演要点、VFX优先级、色温路径

【输出格式】
Markdown表格，按beat分组，每beat一个小标题带导演意图。
```

### 输出
- 一集完整的镜头级分镜（35-55 shots）
- 6-10个关键帧
- 导演笔记

### 为什么这步重要
这是唯一需要"人工精修"的环节。后面所有规则都从这一集里提取。做不好这一集，后面全部走偏。

## Phase 4: 规则归纳 — "把做第一集时的直觉变成规则"

### 输入
- Phase 3 产出的标杆分镜
- 制作人的反馈（如果有调整）

### 要做的事

回顾标杆分镜，回答以下问题并固化为规则文件：

| 维度 | 要回答的问题 | 提取方法 |
|------|---------|---------|
| **场景类型** | 这部剧有哪几种场景类型？各自的镜头风格是什么？ | 从标杆中归纳，结合全剧扫描 |
| **Beat结构** | 平均每场景几个beat？每beat几个shot？ | 从标杆中数 |
| **景别分布** | CU占多少比例？WS呢？ | 从标杆中统计 |
| **角色镜头语法** | 每个主角拍法有什么固定特征？ | 从标杆中总结，对齐角色性格 |
| **提示词模板** | 这部剧的prompt有哪些固定词汇和结构？ | 从标杆中提炼共性 |
| **VFX规则** | 哪些情节触发VFX？VFX的视觉语言是什么？ | 从剧本VFX标注中提取 |
| **关键帧标准** | 什么样的shot值得标为KF？ | 从标杆的KF选择中归纳标准 |
| **质量检查** | 怎么判断一集拆得好不好？ | 从标杆中反推成checklist |

### 实际给Claude的prompt

```
我刚完成了{剧名}第一集的镜头级分镜拆解（附上）。现在请从中提炼出一套可复用的规则，供后续集数批量生产使用。

【需要提取的规则维度】
1. 场景类型分类及各类型的镜头特征
2. Beat划分的逻辑和数据基准（每beat几shot，每集几beat）
3. Shot设计的决策树（什么时候用什么景别/角度/运镜）
4. 角色镜头语法（每个主角的固定拍法）
5. AI提示词的模板结构和标准词汇
6. VFX触发规则和优先级排序方法
7. 关键帧的选择标准
8. 质量检查清单

【同时请输出】
- 一个可以直接丢给Claude新session的"批量生产用系统提示词"
- 把所有规则写在一个md文件中
```

### 输出
- `StoryboardRules.md`（这部剧的专属规则文件）
- 内含批量生产用的Claude系统prompt

---

## Phase 5: 批量生产 — "用规则出剩下的"

### 输入
- StoryboardRules.md
- 剧本文本（按集/按批）
- 已有的角色表/场景库

### 工作方式
```
新session →
  加载 StoryboardRules.md 作为系统上下文 →
  输入5-8集的剧本文本 →
  Claude按规则批量拆 →
  制作人review →
  微调规则（如果需要） →
  继续下一批
```

### 特殊集处理
- **Paywall集：** 最后一shot必须是最强钩子，单独精修
- **VFX重集：** VFX镜头占比会超过20%，需增加导演笔记
- **感情高潮集：** CU占比拉高到60%+，节奏放慢
- **打斗/动作集：** 快切+动态角度为主，shot数量增加

## 整个流程的 Input → Output 一览

```
                    ┌──────────────┐
                    │  完整剧本PDF  │
                    └──────┬───────┘
                           ↓
              Phase 1: 通读理解（30min）
                           ↓
                    ┌──────────────┐
                    │  结构性摘要   │
                    └──────┬───────┘
                           ↓
              Phase 2: 校准问题（5min）
                           ↓
                    ┌──────────────┐
                    │  确认参数     │
                    │  工具/精度/   │
                    │  风格/偏好    │
                    └──────┬───────┘
                           ↓
              Phase 3: 标杆制作（1-2集精拆）
                           ↓
                    ┌──────────────┐
                    │  标杆分镜     │
                    │  Ep01.md     │
                    └──────┬───────┘
                           ↓
              Phase 4: 规则归纳
                           ↓
                    ┌──────────────────┐
                    │  StoryboardRules │
                    │  + 批量prompt    │
                    └──────┬───────────┘
                           ↓
              Phase 5: 批量生产（每批5-8集）
                           ↓
                    ┌──────────────┐
                    │  全部集数分镜  │
                    │  Ep01-64.md  │
                    └──────────────┘
```

## 可复用的核心洞察

1. **规则不是设计出来的，是从样品中归纳出来的。** 先做一集好的，再总结为什么好。不要先写一堆抽象规则再执行。

2. **Beat划分比shot设计更重要。** Beat划对了，shot自然对。Beat划错了，shot再精致也散。

3. **校准问题只问3-4个。** 制作人在看到样品之前，自己也不确定想要什么。问太多 = 浪费时间。

4. **AI提示词的核心是"人物外观描述"。** 因为每个shot独立生成，角色一致性完全靠文字描述锁定。这个描述块必须每shot都带。

5. **导演笔记是给人看的，提示词是给机器看的。** 两者用途不同，不要混淆。提示词描述画面，导演笔记解释"为什么这样拍"。
