# Skill: 情感序列设计师

## 基本信息

- **name**: Emotional_Sequence_Designer
- **版本**: v1.0
- **作者**: 猫叔
- **description**: AI视频情感弧线与情绪节奏精密设计系统
- **关联Skill**: 导演分镜系统、对话场景镜头序列设计、镜头画面设计

## 痛点与解决方案

### 痛点
- 分镜脚本有了镜头和构图，但情绪过渡缺乏精密工程化控制
- 配乐/音效与画面情绪不同步，存在"情绪脱轨"
- 情绪高潮被平均分配，没有张弛有度的呼吸感
- 观众在关键转折点前后缺少足够的情绪铺垫或释放空间
- 多角色的情感线交织时容易互相稀释

### 解决方案
通过系统化的情绪弧线建模、节拍粒度的情感标注、精密的情绪转折卡点设计和声画情绪同步映射，将"感觉对了"转化为可量化、可复现的情感工程方案。

## 核心能力

| 能力模块 | 功能描述 | 适用场景 |
|---------|---------|---------|
| 情绪弧线建模 | 全片/全场景的情绪强度曲线绘制 | 宏观节奏把控 |
| 情绪节拍标注 | 逐镜头的情绪状态、强度、变化向量精密标注 | 微观情感控制 |
| 转折点工程 | 情绪转折前的铺垫链、触发点、余震链设计 | 关键剧情节点 |
| 声画情绪映射 | 将情绪曲线映射为配乐/音效/静默的精确时序 | 音频与画面同步 |
| 呼吸节奏设计 | 紧张→释放→再紧张的张弛周期管理 | 观众注意力管理 |
| 多角色情感交织 | 多条情感线的叠加、冲突与共振设计 | 复杂角色关系 |

## 核心功能

### 1. 情绪弧线建模系统 (Emotion_Arc_Engine)

#### 情绪维度定义

```yaml
Emotion_Dimensions:
  primary_emotion:
    description: "当前镜头的主导情绪"
    values: ["压抑", "悲伤", "孤独", "恐惧", "紧张", "困惑", "震惊",
             "不信", "荒诞", "感动", "温暖", "希望", "坚定", "释放",
             "平静", "庄严", "威严", "脆弱", "倔强", "愤怒"]

  intensity:
    description: "情绪强度 (1-10)"
    scale:
      1-2: "微弱暗流，观众潜意识感知"
      3-4: "可感知的情绪底色"
      5-6: "明确的情绪表达"
      7-8: "强烈的情绪冲击"
      9-10: "情绪顶峰/崩溃点"

  vector:
    description: "情绪变化方向"
    values: ["↑升", "↓降", "→持平", "↗渐升", "↘渐降", "⤴突升", "⤵突降", "∿波动"]

  layer:
    description: "情绪层次（表层 vs 潜层）"
    surface: "角色表面展示的情绪"
    subtext: "角色真实感受但试图压抑的情绪"
    audience: "导演希望观众感受到的情绪（可能与角色表层不同）"
```

#### 情绪弧线模板

```yaml
Emotion_Arc:
  description: "全片或全场景的情绪强度变化曲线"

  arc_types:
    - name: "经典三幕式"
      pattern: "铺垫(2-3) → 渐升(4-5) → 触发(6-7) → 高潮(8-10) → 余震(5-6) → 落点(3-4)"

    - name: "倒V型"
      pattern: "低起(1-2) → 持续攀升(3→8) → 顶峰(9-10) → 急降(4-3) → 留白(2)"

    - name: "阶梯式"
      pattern: "台阶1(3-4) → 平台 → 台阶2(5-6) → 平台 → 台阶3(7-8) → 顶峰(9-10)"

    - name: "波浪式"
      pattern: "波谷(2-3) → 波峰(6-7) → 波谷(3-4) → 更高波峰(8-9) → 最终落点"

    - name: "压抑爆发型"
      pattern: "持续低压(2-3-3-2-3) → 临界(5) → 爆发(9-10) → 释放(4-3)"
```

### 2. 情绪节拍精修系统 (Emotion_Beat_Refiner)

#### 节拍标注模板

```yaml
Emotion_Beat:
  description: "单个情绪节拍的完整标注"

  beat_card:
    shot_range: "S0XX - S0XX"
    beat_name: "节拍名称（如：命运的第一击）"
    primary_emotion: "主导情绪"
    secondary_emotion: "次要/潜层情绪"
    intensity_start: "起始强度 (1-10)"
    intensity_end: "结束强度 (1-10)"
    vector: "变化方向"
    duration_feel: "感知时长（留白/正常/紧凑/急速）"
    anchor_shot: "节拍中的情绪锚点镜头编号"
    breath_type: "呼吸类型（吸气/屏息/呼气/喘息）"
```

#### 节拍间过渡类型

```yaml
Beat_Transitions:
  types:
    - name: "渐变过渡"
      description: "情绪A缓慢融入情绪B"
      technique: "3-5个镜头的渐进过渡"
      example: "压抑→悲伤，通过环境变化和面部微表情递进"

    - name: "硬切转折"
      description: "情绪A被突然事件打断，瞬间切换至情绪B"
      technique: "1个触发镜头完成转折"
      example: "平静→震惊，通过一句话或一个动作瞬间翻转"

    - name: "对比反差"
      description: "故意将两个极端情绪并置"
      technique: "通过场景切换/匹配转场实现"
      example: "街头困苦→豪华酒店，不解释直接跳切"

    - name: "情绪叠化"
      description: "两种情绪在同一镜头中共存"
      technique: "表层/潜层情绪的同时呈现"
      example: "嘴角的笑 + 眼中的泪 = 荒诞的不信"

    - name: "沉默过渡"
      description: "情绪A结束后留白，让观众自行消化，再进入情绪B"
      technique: "插入1-2个空镜或静默反应镜头"
      example: "台词结束后3秒沉默→下一个情绪段落"
```

### 3. 情绪转折点工程 (Turning_Point_Engine)

#### 转折点设计模板

```yaml
Turning_Point:
  description: "情绪转折的完整工程化设计"

  structure:
    pre_chain:
      description: "铺垫链——转折前3-5个镜头的情绪准备"
      elements:
        - "暗示信号：微弱的视觉/听觉线索"
        - "情绪渐变：从A向B缓慢倾斜"
        - "屏息镜头：转折前最后一个'安静'镜头"

    trigger:
      description: "触发点——情绪翻转的精确瞬间"
      elements:
        - "触发类型：台词/动作/声音/画面/沉默"
        - "触发时长：≤2秒"
        - "观众感知：从预期到意外的落差"

    post_chain:
      description: "余震链——转折后3-5个镜头的情绪消化"
      elements:
        - "即时反应：角色的物理反应"
        - "延迟反应：情绪渗透后的行为变化"
        - "新平衡：进入转折后的新情绪基线"
```

### 4. 声画情绪映射系统 (Audio_Emotion_Mapper)

#### 音频情绪映射规则

```yaml
Audio_Emotion_Map:
  description: "将情绪曲线转译为音频指令"

  mapping_rules:
    intensity_1_3:
      music: "环境音为主，配乐极轻或无"
      sfx: "自然音效，低音量"

      music: "单一乐器铺底（钢琴/大提琴）"

    intensity_6_7:
      music: "弦乐组加入，旋律开始发展"
      sfx: "音效与情绪同步加强"
      silence: "精确的短暂静默用于卡点"

    intensity_8_9:
      music: "完整配器，主题旋律全面展开"
      sfx: "戏剧化音效设计"
      silence: "在最高点前1秒的'屏息静默'"

    intensity_10:
      music: "两种选择：完整高潮 或 突然静默（反高潮）"
      sfx: "极端化处理"
      silence: "高潮后的长静默是最有力的音频设计"

  special_techniques:
    - name: "情绪预奏"
      description: "配乐比画面情绪提前2-3秒进入，引导观众情绪"

    - name: "反差静默"
      description: "在情绪最强烈时突然抽走所有声音"

    - name: "音效心理化"
      description: "将角色的心理状态映射为放大的环境音"

    - name: "配乐呼吸"
      description: "配乐的渐强渐弱与角色/观众的呼吸节奏同步"
```

### 5. 情绪呼吸节奏系统 (Emotional_Breathing)

#### 呼吸周期设计

```yaml
Breathing_Cycles:
  description: "张弛有度的情绪呼吸周期管理"

  cycle_structure:
    inhale: "吸气——情绪开始积蓄，紧张度上升"
    hold: "屏息——到达临界点，所有元素悬停"
    exhale: "呼气——情绪释放，紧张度下降"
    rest: "静歇——进入短暂平台，为下一周期准备"

  pacing_rules:
    - "每30秒成片至少包含一个完整呼吸周期"
    - "相邻两个高潮点之间必须有一个'呼气+静歇'段"
    - "全片情绪最高点前必须有最长的一次'屏息'"
    - "全片最后一个呼吸周期的'呼气'应比前几次更长更缓"

  attention_management:
    peak_duration: "单次情绪高峰持续不超过8-10秒"
    valley_function: "情绪低谷不是'无聊'，而是'为下一次冲击充能'"
    rhythm_variation: "相邻呼吸周期的时长应有变化，避免机械重复"
```

### 6. 多角色情感交织系统 (Multi_Character_Emotion)

#### 交织模式

```yaml
Character_Emotion_Weave:
  description: "多角色情感线的关系管理"

  weave_patterns:
    - name: "镜像模式"
      description: "两个角色经历相同情绪但表达方式相反"
      example: "Sky哭泣 vs Adam克制——同样的悲伤，相反的表达"

    - name: "对位模式"
      description: "两个角色同时经历不同甚至对立的情绪"
      example: "一方震惊时另一方平静确认"

    - name: "共振模式"
      description: "两个角色的情绪在某一刻完全同步"
      example: "共同的沉默，共同的泪水"

    - name: "传导模式"
      description: "一个角色的情绪触发另一个角色的情绪变化"
      example: "Sky的问题击穿Adam的铠甲"

    - name: "错位模式"
      description: "角色A还在上一个情绪中，角色B已进入下一个"
      example: "说者已释然，听者刚被击中"
```

## 执行流程

### Step 1: 全局情绪弧线绘制
- 通读完整分镜脚本
- 识别全片的情绪起点、终点
- 标记所有情绪转折点
- 确定全片属于哪种弧线类型
- 绘制情绪强度曲线（镜头编号 vs 强度 1-10）

### Step 2: 场景级情绪分段
- 将全片分割为情绪段落（不一定等于场景）
- 每个段落确定：主导情绪、子弧线类型、强度范围
- 标注段落间的过渡类型

### Step 3: 逐节拍精修
- 以3-8个镜头为一个节拍
- 填写完整的节拍卡
- 标注节拍间过渡

### Step 4: 转折点精密设计
- 为每个情绪转折点设计完整的铺垫链→触发点→余震链
- 确保转折前有足够铺垫，转折后有足够消化

### Step 5: 声画情绪映射
- 将情绪曲线映射为配乐指令时序表
- 标注静默、音效心理化、情绪预奏等特殊技法的精确位置

### Step 6: 呼吸节奏审查
- 检查全片呼吸周期是否合理
- 检查高潮间距是否足够
- 检查留白是否充分
- 检查注意力管理是否有效

### Step 7: 多角色情感交织检查
- 绘制多角色情感线的叠加图
- 检查是否存在情感线互相稀释的问题
- 确认共振点和冲突点的设计

## 与现有Skill对接

### 对接导演分镜系统
- **输入**：分镜脚本的完整表格（8列格式）
- **输出**：情绪精修后的增强标注，可叠加回分镜表格的"情绪&节奏"列和"声音设计"列
- **反馈循环**：如情绪分析发现分镜中的情绪断裂或冗余，反向建议修改分镜

### 对接对话场景镜头序列设计
- 本Skill提供对话段落的情绪上下文
- Dialogue_Shot_Sequence_Designer根据情绪上下文调整反应链强度和正反打节奏

### 对接镜头画面设计
- 情绪强度映射为光影方案：低强度→柔光/灰调，高强度→高对比/戏剧光
- 情绪类型映射为色温：温暖→暖黄，冷峻→冷蓝，混乱→不和谐色

## 参数速查表

| 参数 | 低情绪段 (1-3) | 中情绪段 (4-6) | 高情绪段 (7-9) | 顶峰 (10) |
|-----|--------------|--------------|--------------|----------|
| 平均镜头时长 | 4-6s | 3-4s | 1.5-3s | 0.5-2s |
| 配乐存在感 | 无或极弱 | 单一乐器 | 弦乐组 | 全配器或完全静默 |
| 留白比例 | 40%+ | 20-30% | 10% | 高潮前屏息2-3s |
| 运镜速度 | 极缓/固定 | 缓慢 | 中等/跟随 | 固定（凝视）|
| 剪辑节奏 | 舒缓 | 正常 | 紧凑 | 卡点/冻结 |
